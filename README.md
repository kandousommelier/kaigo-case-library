# 介護の事例ライブラリー

介護現場の「困りごと」を起点に、生産性向上の事例を探すための静的Webアプリです。厚生労働省掲載資料をもとに、伴走支援で使いやすいよう短く要約した事例を収録します。

## 使い方

`index.html` をブラウザで開くだけで利用できます。Webサーバー、ビルド、外部ライブラリは不要です。静的ホスティングでは `cases.json` を読み込み、ローカルファイルとして開いた場合は `app.js` 内の同内容のフォールバックデータを使用します。

## 主な機能

- 困りごと、取組、成果、サービス種別、出典資料名のキーワード検索
- 困りごと、サービス種別、取組分類の複数タグ絞り込み
- 事例カードとモーダルによる詳細表示
- 条件リセット、並び替え、該当件数表示
- スマートフォンとキーボード操作への対応

## データ項目

`cases.json` の各事例は、以下の項目で構成します。ローカルで直接開く使い方も維持する場合は、`app.js` の `FALLBACK_CASES` にも同じ内容を反映してください。

| 項目 | 内容 |
|---|---|
| `id` | 事例番号 |
| `title` | カードに表示する事例タイトル |
| `service` | サービス種別 |
| `problemCategory` | 問題虫めがねと共通の困りごと5分類 |
| `problemDetails` | 困っていたことの短い説明 |
| `categories` | 取組分類タグ |
| `approach` | 取り組んだこと |
| `outcome` | 成果。数値が不明な場合は控えめな表現にする |
| `tip` | 真似できるポイント |
| `suitableFor` | この事例が向いている施設・事業所 |
| `supportUse` | 伴走支援での使い方 |
| `source` | 出典URL。原則としてPDFまたは厚労省ページ |
| `sourceType` | `individual-pdf`、`service-guide-pdf`、`summary-page` など |
| `sourceTitle` | 出典資料名。カードと詳細画面に表示される |
| `sourcePdf` | PDFを直接開くURL。`source` と同じでよい |
| `sourceNote` | ページ番号、抽出時の注意、要約方針など |

## 「問題虫めがね」との共通ルール

- 利用者の課題を「困りごと」という平易な言葉で入口にする
- `problemCategory`、`service`、`categories` の分類語をタグ、カード、詳細で一貫して使う
- 困りごと分類は以下の5つに統一する
  - 物を探すこと、準備、片付け、移動
  - 仕事の流れ、誰が何をするか、仕事の偏り
  - 記録や書類の作成
  - 連絡、申し送り、みんなに伝えること
  - 目標の共有や改善活動
- 青緑を主色、オレンジを注目色とし、角丸の検索欄とタグで探索行動を示す
- 専門用語より現場で使う表現を優先し、成果と「真似できるポイント」を短文で見せる

## PDFから事例を追加する手順

PDF本文からの抽出はCodexに任せず、先にChatGPT等で事例データを作成します。その後、作成済みのJSONを `cases.json` と `app.js` の `FALLBACK_CASES` に反映します。

1. 厚生労働省PDFから、事例タイトル・サービス種別・困りごと・取組・成果を抽出する
2. 本文を長く転載せず、1事例につき短い要約にする
3. `problemCategory` を5分類のいずれかに割り当てる
4. `categories` を2〜4個程度設定する
5. `sourceTitle`、`sourcePdf`、`sourceNote` を必ず設定する
6. `cases.json` に追加する
7. ローカル表示用に `app.js` の `FALLBACK_CASES` にも同じ内容を反映する
8. GitHub Pagesで件数・検索・詳細表示を確認する
